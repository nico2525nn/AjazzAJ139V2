# Ajazz AJ139 V2 リバースエンジニアリングメモ

このファイルには、Web上の制御ソフト(https://139v2mc.yjx2012.com/)のJSコードを解析して得られたプロトコル情報と、これまでのチャットの経緯をメモしています。

## チャットの経緯
1. ユーザーより「Ajazz AJ139 V2」用のオンデバイス制御ソフトを作りたいとの要望。
2. 対象のWebアプリのJSコード(`page-*.js`)を解析し、WebHIDベースで通信が行われていることを確認。
3. `hidapi` を用いてPythonから制御する想定で要件定義し、今後のUIの拡張性を考慮しつつTkinterで仮のGUIを作成することに決定。
4. `pyinstaller` でのビルドを見据え、依存ライブラリの問題を回避しつつビルドするためのバッチファイル(`build.bat`)も用意することに。

## 通信仕様 (HID Protocol)
- **Vendor ID**: `0xA8A4` (43172) または `0xA8A5` (43173)
- **Product ID**: `0x2255` (8789)
- **Usage Page**: `0xFF01` (65281)
- **Usage**: `0x10` (16)
- **Report ID**: `0x00`
- **パケット長**: 64 bytes (システムよってはReport IDを含め65 bytesでやり取りする)

## パケットフォーマット・コマンド一覧
- 送信コマンドは基本的に65バイト配列の先頭から開始します（`buf[0]=0`がReport ID）。
- 常に `buf[1] = 85 (0x55)` で始まり、`buf[2]` にサブコマンドが入ります。

### 1. バージョン情報取得
- **送信:** `[0, 85, 3] + [0]*62`
- **受信:**
  - `buf[23], buf[24], buf[25]` にアスキー文字として記録。（例: `"2.1.2"` -> `[..., 0x32, 0x31, 0x32]`）

### 2. バッテリー情報の取得
- **送信:** `[0, 85, 48, 165, 11, 46, 1, 1, 0, 0, 0] + [0]*54`
- **受信:**
  - `buf[8]`: バッテリー残量 (0%〜100%)
  - `buf[9]`: 充電フラグ (0:未充電, 1:充電中)

### 3. オンライン状態の取得
- **送信:** `[0, 85, 237, 0, 1, 46, 0, 0] + [0]*57`
- **受信:**
  - `buf[8]`: `2` の場合はオンライン（接続済み）。それ以外はオフライン。

### 4. マウス全体設定の取得
- **送信:** `[0, 85, 14, 165, 11, 47, 1, 1, 0, 0, 0] + [0]*54`
- **受信:**
  - `buf[9]`: 現在のLEDモード (Light Mode)
  - `buf[10]-1`: レポートレート・ポーリングレートインデックス
  - `buf[11]`: DPIフラグ数・段数
  - `buf[12]-1`: 現在アクティブなDPIインデックス
  - `buf[13..14]`: DPI 1の値 (リトルエンディアン)
  - `buf[15..16]`: DPI 2の値
  - ... (その他の設定なども同じペイロードに格納される)
