# Ajazz AJ139 V2 リバースエンジニアリングメモ

このファイルは、Web 制御ソフト [https://139v2mc.yjx2012.com/](https://139v2mc.yjx2012.com/) の解析結果と、このリポジトリで実装済みの仕様をまとめたメモです。

## 現在の実装状況

実装済み:
- 基本接続とファームウェア読込
- バッテリー監視
- オンライン判定
- DPI / ポーリング / デバウンス / LOD / ライティング設定
- タスクトレイ表示
- 英語 / 日本語 UI
- 8 ボタンのキーマッピング
- マクロ読込 / 書込 / 初期化
- 32 マクロスロットの UI 管理
- マクロ録画
- マクロイベントの手動追加
- マクロ表の直接編集

未保証:
- すべての個体差に対するマクロ応答形式
- ベンダー固有の特殊マクロモードの完全解釈

## 通信仕様

- Vendor ID: `0xA8A4` または `0xA8A5`
- Product ID: `0x2255`
- Usage Page: `0xFF01`
- Usage: `0x10`
- Report ID: `0x00`
- パケット長: 64 bytes
- Windows / hidapi では 65 bytes で見えることがあり、先頭 `0x00` を落として扱う

## 基本コマンド

### バージョン取得
- 送信: `[0, 85, 3] + [0]*62`

### バッテリー取得
- 送信: `[0, 85, 48, 165, 11, 46, 1, 1, 0, 0, 0] + [0]*54`
- 受信:
  - `buf[8]`: battery
  - `buf[9]`: charging flag

### オンライン状態
- 送信: `[0, 85, 237, 0, 1, 46, 0, 0] + [0]*57`
- 受信:
  - `buf[8] == 2` で online

### 全体設定取得
- 送信: `[0, 85, 14, 165, 11, 47, 1, 1, 0, 0, 0] + [0]*54`

## キーマッピング

### 読込
- 送信: `[0, 85, 8, 165, 11, 32, 0, 0, 0, 0, 0]`

### 書込
- 送信開始: `[0, 85, 9, 165, 34, 49, ...]`

### リセット
- 送信開始: `[0, 85, 9, 165, 34, 32, ...]`

### 形式
- 8 スロット
- 各スロット 4 bytes
  - `type`
  - `code1`
  - `code2`
  - `code3`

### 代表的な型
- `type=16`: 修飾キーコンボ
  - `code1`: modifier bits
  - `code2`: HID keycode
- `type=112`: マクロ割り当て
  - `code1`: macro slot
  - `code2`: repeat count
  - `code3`: vendor mode enum

## マクロ

### 読込
- 送信: `[6, 12, length, offset_lo, offset_hi]`
- 基本単位: 56 bytes
- 全体サイズ: 4096 bytes
- このデバイスでは空チャンク時に応答しないことがある
- 実装では短い待機で空データ扱いし、長時間ブロックしないようにしている

### 書込
- 送信開始: `[0, 85, 13, 0, 0, chunk_len, offset_lo, offset_hi, 0, ...chunk]`
- 最後に commit:
  - `[0, 85, 16, 165, 34, 0, 0, 0, 5]`

### 初期化
- 送信: `[6, 15, 4]`

### データ形式
- 先頭 64 bytes: 32 スロット分の開始オフセット
- 各イベント 4 bytes:
  - `delay_lo`
  - `delay_hi`
  - `flags`
  - `code`

### flags
- `0x40`: press
- `0x80`: end-of-profile
- `0x03`: mouse event
- `0x02`: keyboard event
- `0x04`: other event

## 実装上の重要事項

### 1. 通信競合対策
- コマンド送信前に入力キューを掃除
- `0xAA` とコマンド ID でレスポンスをフィルタ
- `threading.RLock()` で送受信全体を保護

### 2. Report ID ずれ対策
- 65 bytes で返ってきたとき、先頭が `0x00` なら除去して 64 bytes 扱いに統一

### 3. マクロ読込の重さ対策
- 起動時にはマクロを読まず、`Macro` タブを開いたときに遅延読込
- 重い HID 操作は UI スレッド外で実行
- マクロ読込中はタスクトレイのバッテリー監視を一時停止
- 空チャンクでタイムアウトを大量発生させないよう、マクロ読込だけ短い待ち時間を使う

### 4. UI に関するメモ
- `Key Mapping` タブ: 8 ボタン、プリセット、修飾キー、マクロ割り当て
- `Macro` タブ: 32 スロット、録画、イベント編集、状態表示
- `Name` / `Action` / `Delay` はダブルクリック編集対応
- `F13` から `F24` まで選択可能
